## 3. 結果の解釈と考察 (AI/Antigravityによる分析)

### ① 全体的な見かけの精度は「非常に高い」
検証データ（Validation）に対する **R²スコアが `0.881`** というのは、機械学習のベースラインモデル（初回のベースとなるモデル）としては極めて優秀な数値です。モデルが売上のばらつきの約88%を説明できていることを意味します。

### ② 🚨 データリーケージ（情報漏洩）の兆候が顕著
何も工夫をしていない初回実行にもかかわらず、学習時（Train）のR²が `0.963` とほぼ満点に近い数値を叩き出しています。
これは事前手順書での忠告にもあった通り、**「未来を予測する時には本来手に入らないはずのデータ（例：`total_quantity` や `unit_price` など）が学習データに含まれてしまっており、モデルが単なる掛け算の答え合わせ（カンニング）をしている可能性（データリーク）」**の明確な証拠と言えます。この仕組みのまま本番運用に出すことはできません。

### ③ 過学習（Overfitting）の発生
現在の学習データ件数が少ない（約1100件）こともあり、Train の RMSE（約259）に対して Test の RMSE（約1526）が大きく跳ね上がり、テストデータの正答率（Test R²）も `0.708` に急低下しています。
これはモデルが「過去のデータ」の細かいパターンを丸暗記してしまった結果、未知の新しいデータに対して予測を大きく外してしまう**過学習（Overfitting）**が起きている状態です。今回選ばれた `LightGBM` は非常に強力なアルゴリズムですが、データ量が少なく、そのままの特徴量をすべて投入するとこのような過学習を起こしやくなります。

## 4. 次のアクション (Story 4-3に向けて)
現在のベースラインモデルは「不要なデータによるカンニング」で高得点を取っている状態です。
次のステップ（Story 4-3 モデルのカスタマイズ）では、出力された**「生成ノートブック」**のリンクを開き、以下の対応を行う必要があります。

1. **特徴量の重要度（Feature Importance）を確認する**
   - モデルがどのデータを見て売上を予測したのか（`total_quantity`の重要度が異常に高くなっていないか等）を確認する。
2. **不適切なカラムを除外してノートブック上で再学習を行う**
   - 明らかにデータリークを引き起こしているカラムをデータセットから除外し、正しい条件で同じアルゴリズム（LightGBM）を再学習させ、真の実力を測定する。
