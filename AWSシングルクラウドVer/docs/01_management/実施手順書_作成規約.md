# 実施手順書 作成規約

このファイルは、スプリント実施手順書（`sprint*/Story*.md`）を作成・更新する際の統一ルールを定める。

---

## 1. ファイル構成

```
sprint{N}/
├── Story{X}-{Y}～{Z}実施手順書.md   ← 手順書本体（操作手順のみ）
├── notebooks_sprint{N}/              ← 実行コード（Python ノートブック・SQL ファイル等すべて）
│   └── story{X}-{Y}/
│       ├── nb_01_xxx.py
│       ├── nb_02_yyy.py
│       └── 5-1_月別売上推移.sql
└── 証跡/                             ← スクリーンショット・実行ログ
    └── Story{X}-{Y}/
```

> **命名規則**: コードの格納フォルダは必ず **`notebooks_sprint{N}`** とする（`queries/` や `scripts/` 等は使わない）。

---

## 2. コードの外出しルール（最重要）

**実施手順書にコードを埋め込まない。**

コード（SQL・Python・YAML・JSON 等）は必ず外部ファイルに切り出し、手順書からはファイル名で参照する。

### NG 例

```markdown
### Step 2-2: 月別売上推移クエリ

以下の SQL を入力して実行する。

\`\`\`sql
SELECT DATE_TRUNC('month', order_date) AS order_month, ...
\`\`\`
```

### OK 例

```markdown
### Step 2-2: 月別売上推移クエリ

以下のクエリファイルを SQL エディタで開いて実行する。

`notebooks_sprint5/story5-1/5-1_月別売上推移.sql`
```

### 理由

- コードの変更時に手順書も同時更新が必要になり二重管理になる
- Databricks で実際に実行するのはノートブック・SQL ファイルであり、手順書はあくまで操作ガイド
- レビュー・差分管理が容易になる

---

## 3. 手順書に書いてよいもの／書かないもの

| 項目 | 手順書に書く | 外部ファイルに書く |
|------|:-----------:|:-----------------:|
| UI の操作手順（クリック・入力の手順） | ✅ | |
| 設定値の一覧表（テーブル形式） | ✅ | |
| 確認項目・完了チェックリスト | ✅ | |
| 目的・背景の説明文 | ✅ | |
| SQL クエリ | | ✅ `.sql` ファイル |
| Python コード | | ✅ `.py` ノートブック |
| YAML / JSON 設定 | | ✅ 設定ファイル |
| コマンドライン（aws cli, databricks cli 等） | | ✅ シェルスクリプト or README |

---

## 4. フォーマット規約

### 4-1. 概要テーブル（必須）

手順書の冒頭に以下の概要テーブルを入れる。

```markdown
## 概要

| 項目 | 内容 |
|-----|------|
| 対象 Story | X-X, X-X |
| Epic | Epic X: xxxxxxxxxx |
| 目的 | xxxxxxxx |
| 前提 | xxxxxxxx |
```

### 4-2. Phase → Step 構成

```
## Phase N: xxxxx（Story X-X）

### 目的
（このフェーズで何を達成するか）

### Step N-N: xxxxx

（操作手順）
```

### 4-3. 設定値はテーブルで示す

UI 入力欄の値は `| 設定項目 | 値 |` 形式のテーブルで示す。

### 4-4. 言語

基本日本語のみで記述する。
プロダクトに関する専門用語は日英併記。UIの公式訳語が不明なため、両方を参考にしたい。

---

## 5. 証跡

完了チェックリストとスクリーンショットを `証跡/Story{X}-{Y}/` 配下に保存する。
