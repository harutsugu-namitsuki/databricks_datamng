# CloudFormation スタックの更新手順（AWS Console UI編）

既存の環境（RDSのデータやVPCなど）を削除することなく、`cloudformation.yaml` の変更内容（NAT Gatewayの追加など）だけを安全に反映させる「スタックの更新（Update Stack）」の手順です。

---

## 実施手順

### 1. テンプレートファイルの準備
このリポジトリの `開発ドキュメント/cloudformation.yaml` に最新の修正（NAT Gateway等の追加）が反映されている状態であることを確認し、手元のPCに保存しておきます（またはGitHubからダウンロードしておきます）。

### 2. AWS Management Console へのアクセス
1. ブラウザで [AWS Management Console](https://aws.amazon.com/console/) にログインします。
2. 画面上部の検索バーで **CloudFormation** と入力し、CloudFormationのサービスコンソールを開きます。

### 3. 対象スタックの選択と更新の開始
1. 左側のメニューから **スタック** を選択します。
2. 作成済みのスタック（例: `northwind-lakehouse`）をクリックします。
3. 画面右上の **更新** ボタンをクリックします。

### 4. テンプレートの指定
1. 「スタックの更新」画面が開いたら、**既存テンプレートを置き換える** を選択します。
2. 「テンプレートの指定」セクションで **テンプレートファイルのアップロード** を選択します。
3. **ファイルの選択** ボタンを押し、手元にある最新の `cloudformation.yaml` を選択します。
4. 画面右下の **次へ** をクリックします。

### 5. パラメータの確認
1. パラメータの入力画面が表示されます。
2. 以前入力したパラメータ（DBUsername, DBPassword, DatabricksAccountIdなど）はそのまま引き継がれています。**中身は変更せず、そのまま「次へ」** をクリックします。
   > ※ もしパスワード等の再入力を求められた場合は、初期構築時と同じ値を入力してください。

### 6. オプションの設定
1. スタックオプションの設定画面が表示されますが、特に変更は不要です。
2. そのまま画面右下の **次へ** をクリックします。

### 7. レビューと更新の実行
1. 確認画面が表示されます。画面の一番下までスクロールします。
2. 「変更セットのプレビュー」という項目を見ると、「追加されるリソース（Add）」として `NatGatewayEIP` や `NatGateway`、そして「変更されるリソース（Modify）」として `DefaultPrivateRoute` や `DatabricksComputeSG` 等が表示されるはずです。（これにより、既存のRDSやS3が削除・再作成されないことが確認できます）。
3. 一番下の **「AWS CloudFormation によって IAM リソースが作成される場合があることを承認します。」** のチェックボックスにチェックを入れます。
4. **送信**（または スタックの更新）をクリックします。

### 8. 更新完了の確認
1. スタックのステータスが `UPDATE_IN_PROGRESS` になります。
2. NAT Gatewayの作成には数分（2〜3分）かかります。
3. ステータスが緑色の **`UPDATE_COMPLETE`** になれば、更新は成功です！

## 次のステップ
CloudFormationの更新が終わったら、Databricks側で **クラスター（northwind-cluster）を再起動** または 새로アタッチして、Notebook `01_load_northwind_to_rds.py` を再実行してください。
今回はNAT Gatewayがあるため、GitHubからSQLスクリプトをダウンロードし、RDSへ正常にデータを投入できるはずです！
