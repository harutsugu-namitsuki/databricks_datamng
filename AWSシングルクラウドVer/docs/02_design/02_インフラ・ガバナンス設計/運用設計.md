# 運用設計（AWSシングルクラウド版）

## 1. ジョブ実行設計

### 1.1 実行スケジュール

| 項目 | 設定 |
|------|------|
| 実行頻度 | 日次（学習期間は手動、慣れたらJob化） |
| 実行時間 | 毎日 07:00 JST（予定） |
| 実行方式 | Databricks Workflow |
| タイムアウト | 1時間 |

### 1.2 ETLパイプライン構成

> 現状は手動で順次実行。日次バッチ自動化（Databricks Workflow）は今後のスプリントで構築予定。
> `01_load_northwind_to_rds.py` は初回のみ実行（RDSへのデータ投入）のため、日次パイプラインには含まない。

```
ETLパイプライン（手動実行）:
├── Task 1: 02_etl_bronze_ingest.py      ← Bronze取り込み + ingestion_log記録
├── Task 2: 03_etl_silver_transform.py   ← Silver変換 + DQ Check + dq_results記録
├── Task 3: 04_etl_gold_aggregate.py     ← Gold集計 + job_runs記録
└── (将来): Workflow化 + 失敗時SNS通知
```

## 2. 運用テーブル

### 2.1 ops.job_runs（ジョブ実行ログ）

| カラム | 型 | 説明 |
|--------|-----|------|
| run_id | STRING | ジョブ実行ID |
| job_name | STRING | ジョブ名 |
| start_time | TIMESTAMP | 開始時間 |
| end_time | TIMESTAMP | 終了時間 |
| status | STRING | 成功/失敗 |
| executed_by | STRING | 実行者 |
| notebook_path | STRING | 実行ノートブックパス |

### 2.2 ops.ingestion_log（取込ログ）

| カラム | 型 | 説明 |
|--------|-----|------|
| run_id | STRING | ジョブ実行ID |
| table_name | STRING | テーブル名 |
| row_count | LONG | 取得件数 |
| duration_sec | DOUBLE | 所要時間（秒） |
| load_date | DATE | 取り込み基準日 |
| status | STRING | 取り込み結果（✅ / ❌） |

### 2.3 ops.dq_results（品質チェック結果）

| カラム | 型 | 説明 |
|--------|-----|------|
| run_id | STRING | ジョブ実行ID |
| rule_name | STRING | DQルール名 |
| table_name | STRING | 対象テーブル |
| fail_count | LONG | 失敗件数 |
| threshold | LONG | しきい値 |
| result | STRING | PASS/FAIL |
