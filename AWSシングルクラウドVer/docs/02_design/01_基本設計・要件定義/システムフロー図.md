# システムフロー図（AWSシングルクラウド版）

ジョブの実行フローを示すシーケンス図です。

```mermaid
sequenceDiagram
  autonumber
  participant W as Databricks Workflow（自動）
  participant I as 02_Bronze Ingest
  participant P as RDS Postgres（Northwind）
  participant B as Bronze Delta（S3）
  participant T as 03_Silver Transform
  participant S as Silver Delta（S3）
  participant D as DQ Check（03内）
  participant O as Ops Tables（ログ/品質）
  participant G as 04_Gold Aggregate
  participant GM as Gold Delta（S3）

  W->>I: 02_etl_bronze_ingest.py 実行
  I->>P: JDBCでテーブル抽出（VPC内通信）
  I->>B: Bronzeへoverwrite（_load_date等付与）
  I->>O: ingestion_logに記録

  W->>T: 03_etl_silver_transform.py 実行
  T->>S: Silver変換（クレンジング/標準化）
  T->>D: DQルールを実行
  D->>O: dq_resultsに記録

  W->>G: 04_etl_gold_aggregate.py 実行
  G->>GM: Gold（mart 4テーブル）生成
  G->>O: job_runsに完了ログ記録
```

## ETLパイプライン構成

> Databricks Workflow（Job名: `northwind_daily_pipeline`）により毎日 21:00（JST）に自動実行。Job Cluster 使用。実行結果はメール通知済み（Story 1-3〜1-5 完了）。

| Notebook | 実行順序 | 処理内容 | 依存 |
|----------|---------|---------|------|
| `02_etl_bronze_ingest.py` | 1 | RDS → Bronze (S3) + ingestion_log記録 | なし |
| `03_etl_silver_transform.py` | 2 | Bronze → Silver + DQ Check + dq_results記録 | 02完了 |
| `04_etl_gold_aggregate.py` | 3 | Silver → Gold (4マート) + job_runs記録 | 03完了 |
