# システムフロー図（AWSシングルクラウド版）

ジョブの実行フローを示すシーケンス図です。

```mermaid
sequenceDiagram
  autonumber
  participant W as Databricks Workflow/Job（毎日）
  participant I as Ingest（JDBC→Bronze）
  participant P as RDS Postgres（Northwind）
  participant B as Bronze Delta（S3）
  participant T as Transform（Silver/Gold）
  participant D as DQ Check
  participant O as Ops Tables（ログ/品質）
  participant G as Gold Delta（S3）
  participant A as CloudWatch/SNS（アラート）

  W->>I: run開始（run_id, load_date生成）
  I->>P: JDBCでテーブル抽出（VPC内通信）
  I->>B: Bronzeへappend（_load_date等付与）
  I->>O: ingestion_logに記録

  W->>T: Silver/Gold変換を実行
  T->>G: Gold（mart）生成
  T->>D: DQルールを実行
  D->>O: dq_resultsに記録
  D-->>W: OKなら成功 / NGなら失敗

  opt ジョブ失敗時
    W->>A: CloudWatch経由でSNS通知
  end
```

## ジョブ構成

| ジョブ名 | 実行順序 | 処理内容 | 依存 |
|---------|---------|---------|------|
| Ingest Job | 1 | RDS → Bronze (S3) | なし |
| Transform Job | 2 | Bronze → Silver → Gold | Ingest完了 |
| DQ Check | 3 | 品質チェック実行 | Transform完了 |
| Alert | (条件付き) | 失敗時SNS通知 | DQ NG時 |
