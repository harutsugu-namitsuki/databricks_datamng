# 要件定義書（AWSシングルクラウド版）

## 1. プロジェクト概要

### 1.1 プロジェクト名
Northwind データレイクハウス構築プロジェクト（AWSシングルクラウド版）

### 1.2 目的
- Unity Catalog を活用したデータガバナンス基盤の構築
- メダリオンアーキテクチャ（Bronze/Silver/Gold）によるデータ管理
- ETLパイプラインの自動化
- **AWS単一クラウドによるコスト最適化**

### 1.3 設計方針

> **シングルクラウド方針**: 全リソースをAWS上に配置し、クロスクラウド通信コスト・管理複雑性を排除する。

| 項目 | 構成 |
|------|------|
| Databricks環境 | AWS Databricks |
| データレイク | AWS S3 |
| ソースDB | AWS RDS PostgreSQL（同一VPC） |
| 認証 | IAM Role (Instance Profile) |

### 1.4 シングルクラウドを選択した理由

| 観点 | マルチクラウドの問題点 | シングルクラウドのメリット |
|------|---------------------|------------------------|
| **コスト** | クロスクラウドデータ転送料金（AWS→Azure） | VPC内通信で転送料金ゼロ |
| **ネットワーク** | インターネット経由のJDBC接続 | Private Subnet内の低レイテンシ通信 |
| **セキュリティ** | パブリックエンドポイント必須 | VPC内完結（インターネット非公開） |
| **運用** | 2つのクラウドの管理・認証の二重管理 | AWSのみの統一管理 |
| **障害対応** | クラウド間の問題切り分けが困難 | 単一プロバイダで完結 |

## 2. 機能要件

### 2.1 データ取り込み（Ingest）
- [ ] AWS RDS (Northwind) からのJDBC接続によるデータ抽出（VPC内通信）
- [ ] Bronze層へのDelta Lake形式での保存（S3）
- [ ] 日次バッチによるスナップショット取得

### 2.2 データ変換（Transform）
- [ ] Silver層：クレンジング・標準化
- [ ] Gold層：集計・マート作成

### 2.3 データガバナンス
- [ ] Unity Catalog によるメタデータ管理
- [ ] リネージ追跡
- [ ] アクセス制御（スキーマ単位GRANT）

### 2.4 運用管理（Ops）
- [ ] ジョブ実行ログの記録
- [ ] データ品質チェック結果の記録
- [ ] CloudWatch + SNS によるアラート通知

---

## 3. 非機能要件

### 3.1 パフォーマンス
| 項目 | 要件 |
|------|------|
| 日次バッチ処理時間 | 1時間以内 |
| データ量 | 〜10GB/日（学習用のため小規模） |

### 3.2 セキュリティ
| 項目 | 要件 |
|------|------|
| RDS認証方式 | IAM認証 or Databricks Secrets (SSL) |
| S3認証方式 | IAM Role (Instance Profile) |
| データ暗号化 | S3: SSE-S3/SSE-KMS, 転送時: SSL/TLS |
| ネットワーク | VPC内完結（Private Subnet） |

### 3.3 可用性
| 項目 | 要件 |
|------|------|
| 目標稼働率 | 99%（学習用のため緩め） |

## 5. 制約事項

1. 全リソースはAWS `ap-northeast-1` リージョンに配置
2. Databricks ComputeとRDSは同一VPC内のPrivate Subnetに配置
3. S3へのアクセスはIAM Role（Instance Profile）経由のみ
4. 外部インターネットアクセスはNAT Gateway経由
