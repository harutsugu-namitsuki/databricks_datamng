# 環境構築手順書（AWSシングルクラウド版）

## 1. 概要

本書はNorthwindデータレイクハウスの環境構築手順を記載します。
全リソースはAWS上に配置し、VPC内で完結する構成です。

> 参照: [AWSを使用したDatabricksの環境構築 (Flect)](https://cloud.flect.co.jp/entry/2024/10/07/103341)

---

## 2. 前提条件

| 項目 | 要件 |
|------|------|
| AWSアカウント | 有効なアカウント |
| Databricks | AWS Databricks Premium以上（Unity Catalog利用のため） |
| AWS CLI | インストール済み、認証設定済み（GUI操作の場合は不要） |

---

## 3. AWS環境構築

### 3.1 CloudFormationスタック作成

```bash
aws cloudformation create-stack \
  --stack-name northwind-lakehouse \
  --template-body file://cloudformation.yaml \
  --parameters \
    ParameterKey=DBUsername,ParameterValue=dbadmin \
    ParameterKey=DBPassword,ParameterValue=<your-password> \
    ParameterKey=DatabricksAccountId,ParameterValue=<your-databricks-account-id> \
  --capabilities CAPABILITY_NAMED_IAM
```

### 3.2 作成されるリソース

| リソース | 説明 |
|---------|------|
| VPC | 10.0.0.0/16, DNS有効 |
| Public Subnet | NAT Gateway配置 (10.0.0.0/24, AZ-a) |
| Private Subnet 1 | RDS + Databricks Compute (10.0.1.0/24, AZ-a) |
| Private Subnet 2 | Databricks Compute (10.0.2.0/24, AZ-c) — 異なるAZ要件 |
| NAT Gateway | Private Subnetからの外部通信用 |
| S3 VPCエンドポイント | Gateway型 (同一リージョンS3通信最適化) |
| Databricks Compute SG | Egress: 443, 3306, 8443-8451 / Ingress: SG内TCP+UDP |
| RDS SG | Ingress: 5432 from Databricks Compute SG |
| RDS PostgreSQL | Northwindデータベース（Private、パブリックアクセス無効） |
| S3 Bucket | Delta Lake (`s3://lake-northwind-<account-id>/`) + バケットポリシー |
| **カタログIAMロール** | Unity Catalog用 S3アクセス権限 (信頼先: UCMasterRole) |
| **ワークスペースIAMロール** | EC2クラスタ管理権限 (信頼先: 414351767826:root) |
| Secrets Manager | RDS接続情報 |

### 3.3 セキュリティグループ詳細

**Databricks Compute SG (Egress)**:

| ポート | プロトコル | 宛先 | 用途 |
|-------|----------|------|------|
| 443 | TCP | 0.0.0.0/0 | Databricksインフラ、クラウドデータソース、ライブラリ |
| 3306 | TCP | 0.0.0.0/0 | メタストア |
| 8443 | TCP | 0.0.0.0/0 | コントロールプレーンAPI |
| 8444 | TCP | 0.0.0.0/0 | Unity Catalogログ・系列データ |
| 8445-8451 | TCP | 0.0.0.0/0 | 将来的な拡張性 |
| 全ポート | TCP/UDP | SG自身 | 内部トラフィック |

**Databricks Compute SG (Ingress)**:

| ポート | プロトコル | ソース | 用途 |
|-------|----------|-------|------|
| 全ポート | TCP | SG自身 | 内部トラフィック |
| 全ポート | UDP | SG自身 | 内部トラフィック |

### 3.4 S3バケットポリシー

CloudFormationで以下のポリシーが自動設定されます（手動作業不要）:

1. **GrantDatabricksAccess**: Databricksアカウント (414351767826) に Get/Put/Delete/List を許可（`DatabricksAccountId` 条件付き）
2. **PreventDBFSAccessToUnityCatalog**: `unity-catalog/*` パスへのDBFSアクセスを拒否
3. **DenyInsecureTransport**: SSL/TLS以外の通信を拒否

## 5. Databricks設定

### 5.1 Cloud Resources 登録（Account Console）

以下3つの設定を **Cloud resources** から事前登録します。

**① Credentials（資格情報）**

| 項目 | 値 |
|------|-----|
| 名前 | `northwind-credential` |
| IAMロールARN | CloudFormation Output: **`WorkspaceRoleArn`** |

> ⚠️ **ワークスペースロール**（EC2管理用）のARNを指定。カタログロールではない。

**② Storage（ストレージ設定）**

| 項目 | 値 |
|------|-----|
| ストレージ設定名 | `northwind-storage` |
| バケット名 | CloudFormation Output: `S3BucketName` |
| IAMロールARN | CloudFormation Output: **`CatalogRoleArn`** |

> ⚠️ **カタログロール**（Unity Catalog S3アクセス用）のARNを指定。ワークスペースロールではない。

**③ Network（ネットワーク）**

| 項目 | 値 |
|------|-----|
| VPC ID | CloudFormation Output: `VPCId` |
| サブネットID 1 | CloudFormation Output: `PrivateSubnet1Id` |
| サブネットID 2 | CloudFormation Output: `PrivateSubnet2Id` |
| セキュリティグループID | CloudFormation Output: `DatabricksComputeSGId` |

### 5.2 ワークスペース作成

Account Console → Workspaces → Create workspace → **手動 (Custom)**

| 項目 | 設定 |
|------|------|
| ワークスペース名 | `northwind-workspace` |
| リージョン | `Tokyo (ap-northeast-1)` |
| Credential | `northwind-credential` |
| Storage | `northwind-storage` |
| Network | `northwind-network` |

### 5.3 クラスター作成

| 項目 | 設定 |
|------|------|
| アクセスモード | **専用（Single User）** |
| ノードタイプ | 最小構成（学習用） |
| Unity Catalog | 有効 |

### 5.4 Unity Catalog設定

**Step 1: Storage Credential（UIから作成）**

| 項目 | 値 |
|------|-----|
| 名前 | `aws_s3_credential` |
| タイプ | AWS IAM Role |
| IAM Role ARN | CloudFormation Output: `CatalogRoleArn` |

> ⚠️ Storage Credential は **Databricks UIから作成** してください。SQLでは環境依存エラーが発生する場合があります。

**Step 2: カタログ作成（UIまたはSQL）**

```sql
CREATE CATALOG IF NOT EXISTS northwind_catalog;
```

**Step 3: External Location + Schema（SQL — Notebook `00_setup_unity_catalog.py` で実行）**

```sql
-- External Location
CREATE EXTERNAL LOCATION IF NOT EXISTS ext_bronze
  URL 's3://lake-northwind-<account-id>/bronze/'
  WITH (STORAGE CREDENTIAL aws_s3_credential);

-- Schema
CREATE SCHEMA IF NOT EXISTS northwind_catalog.bronze;
CREATE SCHEMA IF NOT EXISTS northwind_catalog.silver;
CREATE SCHEMA IF NOT EXISTS northwind_catalog.gold;
CREATE SCHEMA IF NOT EXISTS northwind_catalog.ops;
```
