# 実装手順書（AWSシングルクラウド版）

本書は、AWSシングルクラウド構成でNorthwindデータレイクハウスを構築する際の実施順序をまとめたものです。

> 参照: [AWSを使用したDatabricksの環境構築 (Flect)](https://cloud.flect.co.jp/entry/2024/10/07/103341)

---

## Phase 0: Databricks アカウントの作成と準備（初回のみ）

AWSの設定を始める前に、Databricks公式サイトからアカウントを作成し、アカウントIDを取得します。

1. [Databricks 無料トライアルページ](https://databricks.com/jp/try-databricks) にアクセスし、クラウドプロバイダーとして **AWS** を選択してアカウントを作成します。
2. アカウント作成後に自動ログイン、もしくは [Databricks Account Console](https://accounts.cloud.databricks.com/) にログインします。
3. 画面右上のユーザー名（またはアイコン）をクリックし、メニュー内にある **Account ID**（英数字の文字列）をコピーして手元に控えます。

> ⚠️ **重要**: このAccount IDは Phase 1 の CloudFormation パラメータ `DatabricksAccountId` として使用するほか、**IAMロール作成時のExternal ID** としても使われます。同一の値です。

## Phase 1: AWS インフラ構築（1回のみ）

### Step 1-1: CloudFormationスタック作成

```bash
aws cloudformation create-stack \
  --stack-name northwind-lakehouse \
  --template-body file://開発ドキュメント/cloudformation.yaml \
  --parameters \
    ParameterKey=DBUsername,ParameterValue=dbadmin \
    ParameterKey=DBPassword,ParameterValue=<your-password> \
    ParameterKey=DatabricksAccountId,ParameterValue=<your-databricks-account-id> \
  --capabilities CAPABILITY_NAMED_IAM
```

> 💡 **既存のスタックを更新する場合**: APIやCLIを使用せず、AWS Console（UI画面）から安全に変更内容だけを反映・アップデートする方法については、別紙 **`開発ドキュメント/CloudFormation更新手順_UI編.md`** を参照してください。

**作成されるリソース**:

| カテゴリ | リソース |
|---------|---------|
| ネットワーク | VPC, Public Subnet, Private Subnet x2, NAT Gateway, IGW, Route Tables, S3 VPCエンドポイント |
| セキュリティ | Databricks Compute SG (egress: 443, 3306, 8443-8451), RDS SG (ingress: 5432) |
| IAMロール | **カタログロール** (Unity Catalog S3アクセス用), **ワークスペースロール** (EC2管理用) |
| データベース | RDS PostgreSQL (Private, パブリックアクセス無効) |
| ストレージ | S3 Bucket (Delta Lake) + バケットポリシー (Databricksアクセス許可 + unity-catalog拒否) |
| シークレット | AWS Secrets Manager (DB接続情報) |

**確認方法**:
```bash
aws cloudformation describe-stacks --stack-name northwind-lakehouse --query "Stacks[0].StackStatus"
# → "CREATE_COMPLETE" になるまで待機
```

**Output取得**:
```bash
aws cloudformation describe-stacks --stack-name northwind-lakehouse --query "Stacks[0].Outputs"
```

以降のステップで使用する値:

| Output Key | 用途 |
|-----------|------|
| `CatalogRoleArn` | Phase 2 Storage設定のIAMロールARN |
| `WorkspaceRoleArn` | Phase 2 Credential設定のIAMロールARN |
| `S3BucketName` | Phase 2 Storage設定のバケット名 |
| `VPCId` | Phase 2 Network設定 |
| `PrivateSubnet1Id` | Phase 2 Network設定 |
| `PrivateSubnet2Id` | Phase 2 Network設定 |
| `DatabricksComputeSGId` | Phase 2 Network設定 |
| `RDSEndpoint` | Phase 4 Secrets設定 |

→ 詳細: `開発ドキュメント/環境構築手順書.md`

## Phase 2: Databricks ワークスペース設定（1回のみ）

### Step 2-1: Databricks Cloud Resources 登録

Account Consoleの左メニュー **Cloud resources** から以下3つを事前登録します。

**① Credentials（資格情報）の登録**

DatabricksがEC2クラスタを管理するための**ワークスペースロール**を登録します。

| 項目 | 値 |
|------|-----|
| 名前 | `northwind-credential` |
| IAMロールARN | CloudFormation Output: **`WorkspaceRoleArn`** |

> ⚠️ **注意**: ここには**ワークスペースロール**（EC2管理用）のARNを指定します。カタログロールではありません。

**② Storage（ストレージ設定）の登録**

Databricksのメタデータを保存するS3バケットと**カタログロール**を登録します。

| 項目 | 値 |
|------|-----|
| ストレージ設定名 | `northwind-storage` |
| バケット名 | CloudFormation Output: **`S3BucketName`** |
| IAMロールARN | CloudFormation Output: **`CatalogRoleArn`** |

> ⚠️ **注意**: ここには**カタログロール**（Unity Catalog S3アクセス用）のARNを指定します。ワークスペースロールではありません。バケットポリシーはCloudFormationで事前設定済みのため手動作業不要です。

**③ Network（ネットワーク）の登録**

| 項目 | 値 |
|------|-----|
| ネットワーク設定名 | `northwind-network` |
| VPC ID | CloudFormation Output: **`VPCId`** |
| サブネットID 1 | CloudFormation Output: **`PrivateSubnet1Id`** |
| サブネットID 2 | CloudFormation Output: **`PrivateSubnet2Id`** |
| セキュリティグループID | CloudFormation Output: **`DatabricksComputeSGId`** |

### Step 2-2: ワークスペースの作成

1. Account Console → **Workspaces** → **Create workspace** → **手動 (Custom)**
2. 以下を入力:
   - **ワークスペース名**: `northwind-workspace`
   - **リージョン**: `Tokyo (ap-northeast-1)`
3. 「既存のクラウドアカウントを使用」を選択し:
   - **Credential**: `northwind-credential` (Step 2-1 ①)
   - **Storage configuration**: `northwind-storage` (Step 2-1 ②)
   - **Network configuration**: `northwind-network` (Step 2-1 ③)
4. **Save** → ステータスが `Running` になれば完了

### Step 2-3: クラスター作成（コスト最小構成）

1. ワークスペースの左メニューから **Compute** をクリックします。
2. 画面右上の **Create compute**（新しいコンピュートを作成する）をクリックします。
3. 画面上部に **UI** / **JSON** の切り替えタブがあります。**方法A（UI）** または **方法B（JSON直貼り）** のどちらかで作成してください。

#### 方法A: UIから設定する場合

画面に表示される項目を上から順に以下の通り設定します:

**【一般セクション】**

| UI項目名 | 設定値 | 理由 |
|----------|--------|------|
| コンピュート名 | 任意（例: `northwind-cluster`） | わかりやすい名前を付ける |
| ポリシー | **制限なし（Unrestricted）** | 学習用途のため制約不要 |

**【ランタイム選択】**

画面に「パフォーマンス」「機械学習」のタブがあります。

| UI項目名 | 設定値 | 理由 |
|----------|--------|------|
| タブ選択 | **パフォーマンス** のまま | 機械学習ランタイムは不要 |
| Databricks Runtime | **最新 LTS** そのまま（例: `17.3 LTS`） | 安定性重視。デフォルトでOK |
| Photonのアクセラレーション | **チェックを外す（OFF）** | Photon有効時はDBU単価が約2倍になるため |

**【ノード構成】**

| UI項目名 | 設定値 | 理由 |
|----------|--------|------|
| 優先ノードタイプ | `rd-fleet.xlarge`（32GB, 4コア）のまま | Databricks管理のFleetインスタンス。デフォルトで最適化されている |
| シングルノード | **ON（有効にする）** | Northwindは小規模データ（全テーブル合計 ~3,000行）。分散処理は不要。ワーカーなしで十分 |
| 非アクティブ状態が X 分継続した後に終了 | **10** 分 | デフォルト値から変更する。アイドル時のコスト削減に重要 |

**【高度なパフォーマンス】** — デフォルトのままでOK（折りたたまれている場合はそのまま）

**【タグ】** — 設定不要（空のまま）

**【アクセスモード】**

| UI項目名 | 設定値 | 理由 |
|----------|--------|------|
| 自動 / 手動 | **手動** に切り替える | アクセスモードを明示的に指定するため |
| アクセスモード | **専用（旧：シングルユーザー）** を選択 | Unity Catalog利用に必要 |
| シングルユーザーまたはグループ | 自分のメールアドレスが自動入力される。そのままでOK | — |

**【AWS固有設定】**

| UI項目名 | 設定値 | 理由 |
|----------|--------|------|
| インスタンスプロファイル | **なし** のまま | CloudFormationでIAMロールを別途設定済み |
| アベイラビリティゾーン | **auto** のまま | AWSが最適なAZを自動選択 |
| ローカルストレージのオートスケールを有効化 | デフォルトのまま | — |
| EBSボリュームのタイプ | **None** のまま | シングルノード学習用途では追加EBS不要 |

4. すべての設定を確認したら、画面下部の **Create compute** をクリックして起動を待ちます（緑色のチェックマークが出れば完了です）。

#### 方法B: JSONで直接入力する場合

画面上部の **JSON** タブをクリックし、以下のJSONを貼り付けます。`single_user_name` は自分のメールアドレスに書き換えてください:

```json
{
  "cluster_name": "northwind-cluster",
  "data_security_mode": "DATA_SECURITY_MODE_DEDICATED",
  "single_user_name": "<あなたのメールアドレス>",
  "kind": "CLASSIC_PREVIEW",
  "spark_version": "17.3.x-scala2.13",
  "node_type_id": "rd-fleet.xlarge",
  "is_single_node": true,
  "runtime_engine": "STANDARD",
  "autotermination_minutes": 10,
  "aws_attributes": {
    "zone_id": "auto"
  }
}
```

貼り付けたら **Create compute** をクリックします。

> 💡 **コスト補足**:
> - **Photon OFF**（`runtime_engine: "STANDARD"`）: Photon有効時のDBU単価は約2倍です。学習用途では不要。
> - **シングルノード**（`is_single_node: true`）: ワーカーノードを起動しないため、マルチノード構成比でEC2コストが大幅に削減されます。
> - **自動終了10分**（`autotermination_minutes: 10`）: アイドル時のコスト浪費を防ぎます。
>
> ⚠️ **「専用（Single User）」とカタログ共有について**:
> 「専用」は**このクラスター上でコードを実行できる人**を1人に限定する設定です。**Unity Catalog のデータ共有には影響しません**。他のユーザーは自分のクラスターや SQL Warehouse から同じカタログ（`northwind_catalog`）にアクセスできます。カタログの権限管理は `GRANT` / `REVOKE` で別途行います。

### Step 2-4: Storage Credential 作成（UIから）

1. ワークスペースの左メニューから **Catalog**（カタログ）をクリックします。
2. 画面上部または左下のメニューから **External Data**（接続/外部データ）→ **Storage Credentials**（資格情報）を開きます。
3. 右上の **Create credential**（資格情報を作成）をクリックします。
4. 以下の通り入力します:
   - **Credential type**（資格情報タイプ）: **AWS IAM Role** を選択
   - **Name**（名前）: `aws_s3_credential`
   - **IAM Role ARN**: Phase 1 の CloudFormation Output **`CatalogRoleArn`** の値を貼り付けます
5. **Create**（作成）をクリックして保存します。

> ⚠️ **SQLではなくUIから作成してください**。SQLでは環境依存エラーが発生する場合があります。
>
> ⚠️ **注意**: ここには**カタログロール**（Unity Catalog S3アクセス用）のARNを指定します。ワークスペースロール（EC2管理用）ではありません。

### Step 2-5: カタログ用 外部ロケーション (External Location) 作成

カタログのマネージドストレージ先として指定できるよう、S3バケット内に専用サブディレクトリの外部ロケーションをUIから作成します。

1. 引き続き、ワークスペース左メニューの **Catalog** → **External Data** を開きます。
2. 今度は **External Locations**（外部ロケーション）タブを開き、右上の **Create location**（ロケーションを作成）をクリックします。
3. 以下の通り入力します:
   - **External location name**: `ext_northwind_catalog`
   - **URL**: `s3://<S3バケット名>/catalog/` （※ Step 1-1で出力された `S3BucketName` を使用。最後に `/catalog/` が必要）
   - **Storage credential**: Step 2-4 で作成した `aws_s3_credential` を選択します。
4. **Create** をクリックして保存します。

### Step 2-6: カタログ作成

**方法A（UIから — 推奨）:**
1. 左メニューの **Catalog** をクリックします。
2. 画面右上の **Create catalog**（カタログを作成）をクリックします。
3. 以下の通り入力します:
   - **Catalog name**: `northwind_catalog`
   - **Type**: `Standard`（デフォルトのまま）
   - **Storage location**: Step 2-5 で作成した **`ext_northwind_catalog`** をプルダウンから選択します。
4. **Create** をクリックします。

**方法B（SQLから）:**

Step 2-3 で作成したクラスターにアタッチした Notebook または SQL エディタで以下を実行します:

```sql
CREATE CATALOG IF NOT EXISTS northwind_catalog
MANAGED LOCATION 's3://<S3バケット名>/catalog/';
```

### Step 2-7: Secrets 設定

RDS接続情報をDatabricksの Secret Scope に登録します。

**Notebook から実行（推奨 — CLI不要）:**

1. ワークスペース左メニューから **+ New** → **Notebook** をクリックして新しいNotebookを作ります。
2. 画面右上のプルダウンから、Step 2-3 で作ったクラスターを選択してアタッチします。
3. セルに以下のコードを貼り付け、`<RDSEndpoint>` と `<パスワード>` をPhase 1でメモした値に書き換えます:

```python
from databricks.sdk import WorkspaceClient

w = WorkspaceClient()

# Secret Scope の作成
w.secrets.create_scope(scope="northwind-secrets")

# RDS接続情報の登録
w.secrets.put_secret(scope="northwind-secrets", key="rds-host",
                     string_value="<RDSEndpoint>")
w.secrets.put_secret(scope="northwind-secrets", key="rds-user",
                     string_value="dbadmin")
w.secrets.put_secret(scope="northwind-secrets", key="rds-password",
                     string_value="<パスワード>")

print("Secretsの登録が完了しました")
```

4. セルの右上にあるプレイボタン（▶）を押して実行します。「Secretsの登録が完了しました」と表示されれば成功です。

> ※ Databricks SDK は Runtime にプリインストール済みのため追加インストール不要です。
> ※ `WorkspaceClient()` は Notebook 上で実行すると自動的に認証されるため、トークンの発行は不要です。
> ※ このNotebookは実行後に削除して構いません。

**登録される Secret 一覧:**

| Key | 値の取得元 |
|-----|----------|
| `rds-host` | CloudFormation Output: `RDSEndpoint` |
| `rds-user` | `dbadmin` |
| `rds-password` | Phase 1 で設定したパスワード |

## 実施順序まとめ

```
Phase 0: Databricks アカウント作成 + Account ID 取得
  └─→ Phase 1: CloudFormation デプロイ (IAMロール2種 + VPC + S3 + RDS)
        └─→ Phase 2: Cloud Resources登録 → ワークスペース作成 → Storage Credential + Secrets
              └─→ Phase 3: 00_setup_unity_catalog.py
                    └─→ Phase 4: 01_load_northwind_to_rds.py
                          └─→ Phase 5: 02 → 03 → 04 (日次繰り返し)
```

| Phase | 実施タイミング | 所要時間目安 |
|-------|-------------|------------|
| 0 | 初回のみ | 〜5分（アカウント作成） |
| 1 | 初回のみ | 〜15分（CloudFormation待ち） |
| 2 | 初回のみ | 〜30分（UI操作含む） |
| 3 | 初回のみ | 〜5分 |
| 4 | 初回のみ | 〜5分 |
| 5 | 日次 | 〜10分 |
