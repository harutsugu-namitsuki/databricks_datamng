# 権限設計（AWSシングルクラウド版）

## 1. 設計方針

**最小権限の原則**を適用し、各レイヤーごとにロールベースのアクセス制御を行う。

> 参照: [AWSを使用したDatabricksの環境構築 (Flect)](https://cloud.flect.co.jp/entry/2024/10/07/103341)

---

## 2. AWS IAM 権限

### 2.1 IAM Role 一覧（2ロール構成）

| IAM Role | 用途 | 信頼先 (Principal) | 主な権限 |
|----------|------|-------------------|---------|
| `northwind-databricks-catalog-role` | Unity Catalog用 (S3メタデータ管理) | `arn:aws:iam::414351767826:role/unity-catalog-prod-UCMasterRole-14S5ZJVKOTYTL` + self-assume | S3 Read/Write/Delete/List |
| `northwind-databricks-workspace-role` | ワークスペース用 (EC2クラスタ管理) | `arn:aws:iam::414351767826:root` + `ec2.amazonaws.com` | EC2 全管理操作 |

> ⚠️ **重要**: この2つのロールは信頼ポリシー (Trust Policy) の Principal が異なるため、**必ず別々に作成** する必要がある。1つのロールに統合すると信頼関係の検証に失敗する。

### 2.2 カタログロール (Trust Policy)

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::414351767826:role/unity-catalog-prod-UCMasterRole-14S5ZJVKOTYTL"
      },
      "Action": "sts:AssumeRole",
      "Condition": {
        "StringEquals": {
          "sts:ExternalId": "<your-databricks-account-id>"
        }
      }
    },
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::<your-aws-account-id>:root"
      },
      "Action": "sts:AssumeRole",
      "Condition": {
        "StringEquals": {
          "AWS:PrincipalArn": "arn:aws:iam::<your-aws-account-id>:role/northwind-databricks-catalog-role",
          "sts:ExternalId": "<your-databricks-account-id>"
        }
      }
    }
  ]
}
```

### 2.3 ワークスペースロール (Trust Policy)

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": "ec2.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    },
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::414351767826:root"
      },
      "Action": "sts:AssumeRole",
      "Condition": {
        "StringEquals": {
          "sts:ExternalId": "<your-databricks-account-id>"
        }
      }
    }
  ]
}
```

### 2.4 Databricks画面でのロール割り当て

| Databricks設定画面 | 指定するロール | 理由 |
|-------------------|--------------|------|
| **Credentials（資格情報）** | ワークスペースロール | EC2クラスタの起動・停止・管理 |
| **Storage（ストレージ設定）** | カタログロール | S3バケットへのメタデータ読み書き |
| **Storage Credential（ワークスペース内）** | カタログロール | Unity CatalogからのS3アクセス |

### 2.5 S3 バケットポリシー

| Statement | 対象 | 効果 | Principal |
|-----------|------|------|-----------|
| GrantDatabricksAccess | `s3://lake-northwind/*` | Allow | `414351767826:root` (DatabricksAccountId条件付き) |
| PreventDBFSAccessToUnityCatalog | `s3://lake-northwind/unity-catalog/*` | Deny | `414351767826:root` |
| DenyInsecureTransport | `s3://lake-northwind/*` | Deny | `*` (非SSL通信) |

### 2.6 External ID について

| 項目 | 値 |
|------|-----|
| External ID | Databricks Account ID (Account Console右上から取得) |
| 用途 | カタログロール・ワークスペースロール両方のTrust Policyで使用 |
| 形式 | `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` (UUID形式) |

> ⚠️ External IDはDatabricksのAWSアカウントID (`414351767826`) ではない。自身のDatabricks Account IDを使用する。

## 3. Unity Catalog 権限

### 3.1 ロール定義

| ロール | 対象者 | 説明 |
|--------|-------|------|
| **Admin** | プロジェクト管理者 | 全権限（カタログ/スキーマ/テーブルの管理） |
| **DataEngineer** | ETL開発者 | Bronze/Silver/Gold/Ops の読み書き |
| **Analyst** | データ分析者 | Gold の読み取りのみ |

### 3.2 スキーマ別アクセスマトリクス

| スキーマ | Admin | DataEngineer | Analyst |
|---------|-------|-------------|---------|
| bronze | ✅ ALL | ✅ ALL | ❌ なし |
| silver | ✅ ALL | ✅ ALL | ❌ なし |
| gold | ✅ ALL | ✅ ALL | ✅ SELECT |
| ops | ✅ ALL | ✅ ALL | ✅ SELECT |

### 3.3 アクセス制限の理由

| 制限 | 理由 |
|------|------|
| Bronze: Analyst不可 | 生データには個人情報やノイズが含まれる可能性 |
| Silver: Analyst不可 | 中間処理データへの直接アクセスは誤解を招く |
| Gold: Analyst読み取り | 提供用に整形されたデータのみ公開 |
| 必要に応じて | Gold上にマスキングViewを作成して公開 |

### 3.4 権限付与SQL

```sql
-- Admin
GRANT ALL PRIVILEGES ON CATALOG northwind_catalog TO admin_group;

-- DataEngineer
GRANT ALL PRIVILEGES ON SCHEMA northwind_catalog.bronze TO engineer_group;
GRANT ALL PRIVILEGES ON SCHEMA northwind_catalog.silver TO engineer_group;
GRANT ALL PRIVILEGES ON SCHEMA northwind_catalog.gold TO engineer_group;
GRANT ALL PRIVILEGES ON SCHEMA northwind_catalog.ops TO engineer_group;

-- Analyst
GRANT USAGE ON CATALOG northwind_catalog TO analyst_group;
GRANT USAGE ON SCHEMA northwind_catalog.gold TO analyst_group;
GRANT SELECT ON SCHEMA northwind_catalog.gold TO analyst_group;
GRANT USAGE ON SCHEMA northwind_catalog.ops TO analyst_group;
GRANT SELECT ON SCHEMA northwind_catalog.ops TO analyst_group;
```
