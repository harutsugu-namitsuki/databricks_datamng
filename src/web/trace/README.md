# 操作トレース可視化システム（ビュー）

自分の操作が内部でどの通信・どのプログラム・どのSQL・どのテーブルを動かしたかを、
リアルタイム（3秒以内）に可視化し、時間窓で通過量を比較する学習ツール。

- 要件定義：[../../../docs/design/app_development/操作トレース可視化システム_要件定義書.md](../../../docs/design/app_development/操作トレース可視化システム_要件定義書.md)
- 実装計画：[../../../docs/plans/2026-06-18-操作トレース可視化システム実装計画.md](../../../docs/plans/2026-06-18-操作トレース可視化システム実装計画.md)

## 起動と使い方

1. 通常どおり API サーバーを起動（リポジトリ直下 `start_store.bat`、または `cd src && uvicorn api.main:app --reload --port 8000`）。
2. ブラウザで **可視化ビュー** を開く：<http://localhost:8000/static/trace/index.html>
3. 別タブで被観測アプリを開く：
   - EC ストア <http://localhost:8000/static/store/login.html>
   - 業務管理 <http://localhost:8000/static/admin/login.html>
4. ビューと並べて、アプリ側でポチポチ操作する。**3秒以内**に経路が点灯し粒子が流れる。

## モード

- **リアルタイム**：操作した1クリックが起こす経路（画面→fetch→CORS→認証→ハンドラ→db→psycopg2→テーブル）を点灯・粒子表示。
  点灯した経路は **次の操作が来るまで粒子の波を繰り返し再生** する（次の操作で最新経路に置き換わる）。
  失敗（認証失敗401・DBエラー）は **赤** で表示（FR-U8）。
- **履歴**：時間窓（10分/1時間/1日/1週/1月/1年）で各経路の通過量を線の太さ＋ヒート色で表示。

ノード／エッジをクリックすると、IT初心者向けの解説（一言／やさしい説明／実体・実SQL／用語注釈）が出る（FR-U7）。

## 仕組み（データの流れ）

```
ブラウザ(trace-client.js: window.fetchにX-Trace-Id付与)
  → FastAPI(ASGIミドルウェアでhttp span / _require_tokenでauth span / db.pyでsql span)
  → trace_data/events-YYYYMMDD.jsonl 追記 ＋ SSE配信(/trace/stream)
  → ビュー（/trace/map で地図、/trace/stream でリアルタイム、/trace/rollup で履歴）
```

- 保存は **JSONLのみ**（`trace_data/` は gitignore）。履歴はロールアップ（地図エッジ別・分/時/日バケット）をメモリ集計し、起動時に当日までのイベントから復元。
- 計測は**観測の副作用ゼロ**：トレース記録の失敗は被観測アプリを止めない（FR-T6／NFR-R1）。

## 受け入れ確認（要件トレース）

| 要件 | 確認方法 |
|---|---|
| NFR-P1 3秒以内 | 操作→点灯までが体感3秒以内 |
| FR-T1 全通信対象 | ページロードの自動通信もビューに出る |
| FR-T3 認証 | 公開API=認証ノードを通らない／保護API成功=緑・失敗=赤 |
| FR-T4 テーブル抽出 | sql のテーブルが正しい／不明時「不明」 |
| FR-U2/U3 履歴・切替 | 6窓で太さ/色が変化 |
| FR-U7 解説 | ノード詳細が4段＋用語注釈 |
| FR-U8 失敗赤 | 401で認証ノードが赤点灯 |
| NFR-R1 副作用ゼロ | `trace_data/` を読取専用にしてもアプリが正常動作 |

## 既知の制限（MVP）

- ロールアップはメモリ保持＋起動時に保持中イベント（既定7日）から復元。長期窓（1月/1年）は実データが貯まるほど充実する。
- 経路マップは事前定義（`src/api/trace_map.json`）。代表5ルート以外のエンドポイント（categories 等）はハンドラノードを持たず、fetch→CORS→db→psycopg2→テーブルのみ点灯する。
