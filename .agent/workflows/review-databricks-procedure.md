# Databricks 実施手順書レビュー スキル

このスキルは、Databricks の実施手順書（`.md`）および関連コード（`.py`, `.sql`）をレビューして不都合・事故リスクを洗い出す際に従うべきチェックリストです。

## いつ使うか

- ユーザーから実施手順書のレビューを依頼されたとき
- 新しい Sprint の手順書が追加されたとき
- 「この手順書通りに進めて問題ないか？」と聞かれたとき

## レビュー手順

### Step 1: 手順書（`.md`）を読む

まず対象の手順書ファイルを `view_file` で全文読み込む。

### Step 2: 関連コードを全て読む

手順書内で参照されている全てのノートブック（`.py`）およびクエリ（`.sql`）を `view_file` で全文読み込む。手順書に「ファイル一覧」等のテーブルがあればそこからパスを取得する。

### Step 3: 以下の「事故パターン」を1つずつチェックする

#### 🔥 事故パターン1: カタログ名・スキーマ名のハードコード
- コード内の `FROM` 句や変数定義にカタログ名（例: `northwind_catalog`）がリテラル文字列で直書きされていないか
- 環境差異（dev / staging / prod）で一括置換が必要になるリスクを指摘する
- **深刻度: 高**（全ファイルが `TABLE_OR_VIEW_NOT_FOUND` で即死する）

#### 🔥 事故パターン2: Databricks UIの世代ギャップ
- 手順書のUI操作手順が、**現在の Databricks UI と一致しているか**を確認する
- 特に以下のポイントは頻繁に変わるため要注意:
  - **ダッシュボード**: 「Classic Dashboard」vs「AI/BI Dashboard（旧 Lakeview）」。2024年以降のワークスペースではデフォルトが AI/BI になっているため、手順書が Classic 前提だと操作が全く合わない
  - **クラスターのランタイム**: Standard Runtime と ML Runtime の違い。AutoML は ML Runtime が**必須**
  - **クラスターのアクセスモード**: 「Single User」「Shared」「No Isolation Shared」の違い。AutoML や MLflow は Single User が安全
  - **メニュー名の変更**: 「Compute」↔「コンピューティング」、「Experiments」↔「実験」など、言語設定やバージョンで表記が異なる

#### ⚠️ 事故パターン3: データリーケージ（ML系手順書のみ）
- 学習データに「予測時には知り得ない情報」が特徴量として含まれていないか
- 時系列データで `automl.regress()` / `automl.classify()` を使っている場合、`time_col` 引数が指定されているか（未指定だとランダム分割＝未来のデータで過去を学習する）
- **深刻度: 中～高**（モデルの精度が実用時に激しく劣化する）

#### ⚠️ 事故パターン4: Pandas変換（`.toPandas()`）によるOOMリスク
- PySpark DataFrame を `.toPandas()` で変換している箇所がないか
- 学習用の小規模データなら問題ないが、本番規模のデータで実行するとドライバのメモリが枯渇してクラスタが即死する
- **深刻度: 低～中**（データ量が小さければ問題なし、大きければ即死）

#### ⚠️ 事故パターン5: SQL Warehouse / クラスターの課金リスク
- SQL Warehouse や All-Purpose クラスターの自動停止（Auto Stop）が設定されているか手順書に記載があるか
- 設定がない場合、作業後に停止し忘れると**数時間〜数日分の課金が発生する**
- **深刻度: 中**（技術的には壊れないが、お金が飛ぶ）

### Step 4: 出力フォーマット

レビュー結果は以下のファイル名で、対象手順書と同じディレクトリに保存する:

```
{対象手順書ファイル名}_Antigravityからの忠言.md
```

ファイル構成は以下のとおり:
1. **🚨 事故レベルの問題**（手順書通りに進めると確実にエラーや不整合が発生するもの）
2. **💡 留意事項**（今回は大丈夫だが、環境や規模が変わると問題になりうるもの）
3. **💻 コードに対する忠言**（ノートブックやSQLに対する改善提案）

Sprint をまたいで共通する事故パターンがある場合は、上位ディレクトリに共通ファイルも作成する。
