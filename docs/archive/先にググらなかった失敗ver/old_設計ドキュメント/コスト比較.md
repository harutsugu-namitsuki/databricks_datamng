# コスト比較：マルチクラウド vs シングルクラウド

## 1. 概要

本ドキュメントは、マルチクラウド構成（AWS+Azure）からAWSシングルクラウド構成への移行によるコスト削減効果を整理したものです。

---

## 2. アーキテクチャ比較

| 項目 | マルチクラウドVer | シングルクラウドVer |
|------|------------------|-------------------|
| Databricks | Azure Databricks | AWS Databricks |
| データレイク | Azure ADLS Gen2 | AWS S3 |
| ソースDB→Databricks | インターネット経由 (JDBC) | VPC内通信 (JDBC) |
| Databricks→ストレージ | Azure内 (Managed Identity) | AWS内 (IAM Role) |
| クラウドプロバイダ数 | **2** (AWS + Azure) | **1** (AWS) |

---

## 3. コスト比較

### 3.1 データ転送コスト

| 項目 | マルチクラウド | シングルクラウド | 差分 |
|------|-------------|---------------|------|
| RDS→Databricks（JDBC） | AWS→Azure間転送（$0.09/GB） | VPC内通信（**無料**） | **削減** |
| Databricks→ストレージ | Azure内通信（低コスト） | AWS内通信（低コスト） | 同等 |
| NAT Gateway | 不要 | 必要（$0.045/h + $0.045/GB） | 微増 |

### 3.2 サービスコスト

| 項目 | マルチクラウド | シングルクラウド | 差分 |
|------|-------------|---------------|------|
| Databricks | Azure DBU料金 | AWS DBU料金 | ほぼ同等 |
| ストレージ | ADLS Gen2 | S3 Standard | ほぼ同等 |
| RDS | AWS RDS | AWS RDS | 同一 |

### 3.3 管理・運用コスト

| 項目 | マルチクラウド | シングルクラウド | 備考 |
|------|-------------|---------------|------|
| クラウドアカウント数 | 2（AWS + Azure） | 1（AWS） | 管理工数半減 |
| 認証管理 | IAM + Azure AD の二重管理 | IAMのみ | 運用簡素化 |
| 障害調査 | クロスクラウドで複雑 | 単一クラウドで完結 | 対応時間短縮 |
| セキュリティ監査 | 2つのクラウドで二重 | 1つに統一 | 監査コスト半減 |

---

## 4. 月額コスト概算（学習規模）

| 項目 | マルチクラウド | シングルクラウド |
|------|-------------|---------------|
| AWS RDS (db.t3.micro) | ~$15/月 | ~$15/月 |
| Azure Databricks | ~$30/月 | − |
| AWS Databricks | − | ~$30/月 |
| Azure ADLS Gen2 | ~$5/月 | − |
| AWS S3 | − | ~$3/月 |
| データ転送（AWS→Azure） | ~$5/月 | **$0** |
| Azure サブスク管理 | 管理コスト有 | **$0** |
| **合計** | **~$55/月 + Azure管理コスト** | **~$48/月** |

> ※ 学習用の小規模データのため差額は小さいが、本番規模ではデータ転送コストの差が大きくなる

---

## 5. 結論

| 観点 | 評価 |
|------|------|
| **直接コスト** | シングルクラウドが安い（データ転送料金ゼロ） |
| **管理コスト** | シングルクラウドが安い（1つのクラウドのみ） |
| **スケーラビリティ** | データ量増加時にシングルクラウドの優位性が拡大 |
| **セキュリティ** | シングルクラウドが優位（VPC内完結） |
| **技術的シンプルさ** | シングルクラウドが優位（クロスクラウド考慮不要） |

**→ AWSシングルクラウドを採用**
