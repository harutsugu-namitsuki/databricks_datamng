# Northwind アプリケーション システム設計

## 作るもの

Northwind データベース (RDS PostgreSQL) を操作する **2つの Web アプリ** を Streamlit で構築する。

| アプリ | 用途 | ユーザー |
|--------|------|----------|
| **業務管理アプリ** | 商品・在庫・従業員のマスタ管理 | 社内の業務担当者 |
| **購買アプリ** | 商品閲覧・カート・注文 | 顧客 (B2B バイヤー) |

両アプリとも RDS に直接読み書きする。書き込まれたデータは日次 ETL パイプライン (Bronze → Silver → Gold) でデータレイクハウスに反映される。

## アーキテクチャ

```
┌─────────────────┐     ┌─────────────────┐
│  業務管理アプリ   │     │   購買アプリ      │
│  (Streamlit)    │     │  (Streamlit)    │
│  :8501          │     │  :8502          │
└────────┬────────┘     └────────┬────────┘
         │                       │
         └───────────┬───────────┘
                     │ psycopg2
                     ▼
         ┌───────────────────────┐
         │  RDS PostgreSQL       │
         │  northwind DB         │
         │  (private subnet)     │
         └───────────┬───────────┘
                     │ JDBC (日次ETL)
                     ▼
         ┌───────────────────────┐
         │  Databricks           │
         │  Bronze→Silver→Gold   │
         └───────────┬───────────┘
                     │
                     ▼
         ┌───────────────────────┐
         │  BI ダッシュボード      │
         └───────────────────────┘
```

## 技術スタック

| レイヤ | 技術 |
|--------|------|
| フロントエンド + バックエンド | Streamlit (Python) |
| DB ドライバ | psycopg2-binary |
| DB | RDS PostgreSQL (既存 Northwind) |
| 認証 | streamlit-authenticator (YAML ベース、初期は簡易実装) |
| デプロイ (将来) | Docker → AWS App Runner or ECS Fargate |

## データモデル (操作対象テーブル)

### 業務管理アプリが操作するテーブル

| テーブル | 操作 | 用途 |
|----------|------|------|
| products | CRUD | 商品マスタ管理 |
| categories | R | 商品カテゴリ参照 |
| suppliers | R | 仕入先参照 |
| employees | RU | 従業員情報閲覧・編集 |

### 購買アプリが操作するテーブル

| テーブル | 操作 | 用途 |
|----------|------|------|
| products | R (+ units_in_stock を UPDATE) | 商品一覧・在庫減算 |
| categories | R | カテゴリ絞り込み |
| orders | C, R | 注文ヘッダ作成・履歴参照 |
| order_details | C, R | 注文明細作成・履歴参照 |
| customers | R | 顧客情報 (ログインユーザー) |
| shippers | R | 配送業者選択 |

## 機能一覧

### 業務管理アプリ

| # | 機能 | 内容 |
|---|------|------|
| A1 | ログイン | YAML ベースの簡易認証 |
| A2 | ダッシュボード | 在庫サマリ、最近の注文、在庫アラート |
| A3 | 商品管理 | 一覧表示・新規登録・編集・廃止フラグ切替 |
| A4 | 在庫管理 | 在庫数の手動調整、発注点アラート表示 |
| A5 | 従業員管理 | 従業員一覧・詳細表示・基本情報編集 |

### 購買アプリ

| # | 機能 | 内容 |
|---|------|------|
| S1 | ログイン | 顧客ID ベースの簡易認証 |
| S2 | 商品一覧 | カテゴリ別表示、検索、価格・在庫表示 |
| S3 | 商品詳細 | 商品情報、数量選択、カート追加 |
| S4 | カート | カート内容表示、数量変更、削除、合計計算 |
| S5 | 注文確定 | 配送先・配送業者選択 → orders + order_details INSERT |
| S6 | 注文履歴 | 過去の注文一覧と明細表示 |

## データ連携フロー

```
[業務アプリ] 商品登録 ──→ RDS products INSERT
                                  │
                              (日次 ETL)
                                  │
                                  ▼
                          Bronze → Silver → Gold
                                  │
                                  ▼
                          BI ダッシュボードに新商品が出現

[購買アプリ] 注文 ──→ RDS orders/order_details INSERT
                       + products.units_in_stock UPDATE
                                  │
                              (日次 ETL)
                                  │
                                  ▼
                          Gold 売上集計が更新
                          在庫が発注点以下 → アラート
```

## セキュリティ

- DB 認証情報は `.env` ファイルまたは環境変数で管理 (リポジトリには含めない)
- 本番では AWS Secrets Manager から取得
- 従業員の個人情報 (生年月日、自宅電話番号等) は権限に応じてマスキング表示
