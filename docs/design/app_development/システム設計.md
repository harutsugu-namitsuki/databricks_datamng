# Northwind アプリケーション システム設計

> **実装方式: FastAPI + HTML/CSS/JS（現行）**
> 旧 Streamlit 版（`:8501`/`:8502`）は廃止し [`docs/archive/旧Streamlitアプリ/`](../../archive/旧Streamlitアプリ/) へ退避済み。
> 本書は現行実装（[`src/`](../../../src/)）を正とする。経緯は [なぜ画面がないの？の回答.md](なぜ画面がないの？の回答.md) を参照。

## 作るもの

Northwind データベース（RDS PostgreSQL）を操作する **2 つの Web アプリ**を、
**1 つの FastAPI サーバー**が配信・仲介する。

| アプリ | 用途 | ユーザー | 入口 URL |
|--------|------|----------|----------|
| **EC ストア**（購買） | 商品閲覧・カート・注文 | 顧客（B2B バイヤー） | `/static/store/login.html` |
| **業務管理**（admin） | 商品・在庫・従業員のマスタ管理 | 社内の業務担当者 | `/static/admin/login.html` |

両アプリとも RDS に読み書きし、書き込まれたデータは日次 ETL（Bronze → Silver → Gold）で
データレイクハウスに反映される。

## アーキテクチャ

図: [アプリ層アーキテクチャ.puml](アプリ層アーキテクチャ.puml)（PlantUML）

```
顧客 / 社内担当者
      │
      ▼
ブラウザ (HTML/CSS/JS, web/store・web/admin)
      │  api.js が fetch で HTTP リクエスト
      ▼
FastAPI サーバー (localhost:8000 / uvicorn)
  ├── /static …… HTML/CSS/JS を静的配信（src/web）
  ├── /api/auth …… 認証（admin=固定アカウント / store=customers照合）
  ├── /api/… …… 業務API・EC API
  └── db.py (psycopg2 / .env)
      │
      ▼
RDS PostgreSQL (northwind / private subnet)
      │  JDBC（日次 ETL）
      ▼
Databricks  Bronze → Silver → Gold  → BI / 分析（別設計書）
```

ポイント：**ブラウザは DB に直接つなげない**ため、FastAPI が「画面の配信」と「DB との仲介（通訳）」の
両方を担う。HTML を file:// で直接開いても動かない（`/api`・`/static` のパスが解決できない）。

## 技術スタック

| レイヤ | 技術 |
|--------|------|
| フロントエンド | 素の HTML / CSS / JavaScript（`fetch` API） |
| バックエンド | FastAPI（Python） |
| DB ドライバ | psycopg2 |
| DB | RDS PostgreSQL（既存 Northwind） |
| 認証 | トークン方式（`Authorization: Bearer`、セッションはサーバーのメモリ保持） |
| 設定 | `.env`（リポジトリ直下、`load_dotenv()` で読込） |
| 配信 | uvicorn（単一プロセス・ポート 8000） |
| デプロイ（将来） | Docker → AWS App Runner / ECS Fargate |

## 認証方式（重要：admin と store で非対称）

| アプリ | 認証元 | 資格情報 | DB 接続の要否 |
|--------|--------|----------|---------------|
| 業務管理(admin) | `main.py` 内の固定アカウント（メモリ） | `admin`/`admin123`, `staff`/`staff123` | **不要**（DB 無しでもログイン可） |
| EC ストア(store) | DB の `customers` テーブル（91 社） | 顧客ID、パスワード＝顧客ID（大小文字不問） | **必須**（未接続・データ未ロードだと誰もログイン不可） |

- 認証成功でサーバーがトークンを発行し、メモリ上の `_sessions` に保持。以降は `Bearer` トークンで認可。
- EC ストアに専用のアカウント表は無い。**顧客マスタがそのままログインアカウント**。

## データモデル（操作対象テーブル）

### 業務管理アプリ

| テーブル | 操作 | 用途 |
|----------|------|------|
| products | CRUD | 商品マスタ管理 |
| categories / suppliers / shippers | R | 参照（プルダウン等） |
| products.units_in_stock | U | 在庫調整（`/api/inventory/adjust`） |
| employees | R / U | 従業員閲覧・編集（PII は権限でマスキング） |

### EC ストアアプリ

| テーブル | 操作 | 用途 |
|----------|------|------|
| products | R（+ units_in_stock を UPDATE） | 商品一覧・注文時の在庫減算 |
| categories | R | カテゴリ絞り込み |
| orders / order_details | C, R | 注文の作成・履歴参照 |
| customers | R | ログインユーザー情報 |
| shippers | R | 配送業者選択 |

## API 一覧（[src/api/main.py](../../../src/api/main.py)）

| メソッド | パス | 用途 | 認可 |
|---|---|---|---|
| POST | `/api/auth/admin` | 業務管理ログイン | - |
| POST | `/api/auth/store` | EC ストアログイン | - |
| GET | `/api/products` | 商品一覧（カテゴリ/検索フィルタ） | - |
| POST / PUT | `/api/products` `/api/products/{id}` | 商品 登録 / 更新 | admin |
| GET | `/api/categories` `/api/suppliers` `/api/shippers` | マスタ参照 | - |
| POST | `/api/inventory/adjust` | 在庫調整 | admin |
| GET / GET | `/api/employees` `/api/employees/{id}` | 従業員 一覧 / 詳細（PII マスキング） | admin |
| PUT | `/api/employees/{id}` | 従業員 更新 | admin |
| POST | `/api/orders` | 注文確定（orders + order_details + 在庫減算） | store |
| GET | `/api/orders` `/api/orders/{id}/details` | 注文履歴 / 明細 | store |
| GET | `/api/dashboard` | KPI・最近の注文・在庫アラート | admin |

## 機能一覧

### 業務管理アプリ

| # | 機能 | 内容 |
|---|------|------|
| A1 | ログイン | 固定アカウント（admin/staff） |
| A2 | ダッシュボード | 在庫サマリ、最近の注文、在庫アラート |
| A3 | 商品管理 | 一覧・新規登録・編集・廃止フラグ切替 |
| A4 | 在庫管理 | 在庫数の手動調整、発注点アラート表示 |
| A5 | 従業員管理 | 一覧・詳細・基本情報編集（PII は権限でマスキング） |

### EC ストアアプリ

| # | 機能 | 内容 |
|---|------|------|
| S1 | ログイン | 顧客ID 認証（customers 照合） |
| S2 | 商品一覧 | カテゴリ別・検索・価格/在庫表示 |
| S3 | 商品詳細 | 商品情報、数量選択、カート追加 |
| S4 | カート | 内容表示・数量変更・削除・合計（localStorage 保持） |
| S5 | 注文確定 | 配送先・配送業者選択 → orders + order_details INSERT |
| S6 | 注文履歴 | 過去の注文一覧と明細表示 |

詳細な画面構成は [画面設計.md](画面設計.md)、データ連携は [E2Eアプリ連携設計書.md](E2Eアプリ連携設計書.md) を参照。

## データ連携フロー

```
[業務管理] 商品登録 ──→ RDS products INSERT
                              │ 日次 ETL
                              ▼  Bronze → Silver → Gold
                          BI ダッシュボードに新商品が出現

[EC ストア] 注文 ──→ RDS orders / order_details INSERT
                     + products.units_in_stock UPDATE
                              │ 日次 ETL
                              ▼
                          Gold 売上集計が更新／在庫が発注点以下 → アラート
```

## セキュリティ

- DB 認証情報は `.env`（リポジトリ直下、git 管理外）。本番は AWS Secrets Manager から取得。
- 従業員の個人情報（生年月日・自宅電話番号等）は、`/api/employees/{id}` で **admin 以外はマスキング**して返す。
- CORS は開発中は全許可。本番では Origin を限定する。
- 現状の認証は簡易実装（平文比較・メモリセッション）。本番化時はハッシュ化・有効期限・永続セッションが必要。
