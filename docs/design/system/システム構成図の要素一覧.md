## ステータス凡例

| ステータス | 意味 |
|------------|------|
| ✅ 記載済み | 図に明示的に記載 |

## 1. ネットワーク層

| 要素 | 説明 | ステータス | 備考 |
|------|------|------------|------|
| **VPC** | Databricks ComputeとRDSを同一VPCに配置 | ✅ | CIDR: 10.0.0.0/16 |
| **Private Subnet 1** | Compute + RDS配置 (AZ-a) | ✅ | 10.0.1.0/24 |
| **Private Subnet 2** | Compute配置 (AZ-c) — Databricks要件: 異なるAZ | ✅ | 10.0.2.0/24 |
| **Public Subnet** | NAT Gateway配置 | ✅ | 10.0.0.0/24 |
| **Security Groups** | Compute SG (Egress: 443, 3306, 8443-8451, Ingress: SG内TCP/UDP) + RDS SG (Ingress: 5432 from Compute) | ✅ | |
| **IGW** | VPCからインターネットへの出口 | ✅ | CF: InternetGateway |
| **ルートテーブル** | Public RT (IGW経由) + Private RT (NAT経由) | ✅ | CF: PublicRouteTable + PrivateRouteTable |
| **Network ACL** | サブネットレベルのファイアウォール | 🔧詳細設計 | |
| **NAT Gateway** | Private SubnetからのOutbound接続 | ✅ | EIP付き |
| **VPC Endpoint (S3)** | S3へのプライベート接続（Gateway型） | ✅ | CF: S3VPCEndpoint |

## AWSシングルクラウド構成の特徴

| 特徴 | 内容 |
|------|------|
| **Databricks環境** | AWS上（同一VPC内） |
| **RDSアクセス** | Private Subnet + VPC内通信 |
| **S3認証** | IAM Role（カタログロール: UCMasterRole信頼 + ワークスペースロール: クロスアカウント） |
| **セキュリティレベル** | 高（VPC内完結） |
| **コスト** | 低（クロスクラウド転送なし） |
