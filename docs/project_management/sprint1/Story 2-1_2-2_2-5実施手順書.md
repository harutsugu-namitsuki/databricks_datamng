## Phase 2: グループ作成と権限設定（Story 2-2 最小版）

すべての SQL は **admin アカウント** でログインしたノートブックで実行する。

### Step 2-1: テストユーザーの招待

1. Databricks UI → **Settings** → **Identity and access** → **Users** → **Add user**
2. テスト用メールアドレスを入力して招待（本番アカウントとは別の Google / Microsoft アカウント推奨）
3. 招待メールから初回ログインを完了させる

> **補足**: テスト完了後にユーザーを削除しても構わない。

### Step 2-2: Unity Catalog グループの作成

Databricks ではグループ作成は **UI で行う**（SQL コマンド不可）。

1. **Settings** → **Identity and access** → **Groups** → **Add group**
2. Group name に `analyst_group` を入力して **Save**

### Step 2-3: グループへのユーザー追加

1. **Settings** → **Identity and access** → **Groups** → `analyst_group` をクリック
2. **Members** タブ → **Add member**
3. Step 2-1 で招待したテストユーザーのメールアドレスを選択して追加

### Step 2-4 〜 Step 2-5: アクセス権の付与と確認

**目的:** `analyst_group` に「Gold 層だけ見られる」権限を与える。

GRANT とは「このグループにこの操作を許可する」という SQL コマンド。Unity Catalog はデフォルトで何も許可しない（触れない）ため、必要な範囲だけを明示的に解放する必要がある。

- Bronze / Silver には何も付与しない → 自動的にアクセス拒否
- Gold には USE（スキーマに入る）と SELECT（テーブルを読む）を付与

Step 2-2 でグループを作成済みであることを確認してから実行すること（グループが存在しないと `PRINCIPAL_DOES_NOT_EXIST` エラーになる）。

`notebooks/story2-1~2-5/nb_01_grants.sql` を **admin アカウント**のノートブックで実行する。

- セル1: カタログへの USE 権限（Gold スキーマに辿り着くための入口）
- セル2: Gold スキーマへの USE + SELECT 権限（実際にデータを読む権限）
- セル3: `SHOW GRANTS ON SCHEMA northwind_catalog.gold` で確認 → `analyst_group` に `USE SCHEMA（スキーマ内のオブジェクトを表示する。）` と `SELECT（テーブルやビューからデータを読み取る。）` が表示されれば成功

## Phase 3: アクセス制御の証明（Story 2-5）

**テストユーザーのアカウント**でログインして実行する（シークレットモード推奨）。

### Step 3-1: テストユーザーでログイン

1. ブラウザのシークレットモードを開く
2. テストユーザーのアカウントで Databricks にログイン
3. All-Purpose クラスターに接続したノートブックを新規作成

### Step 3-2: Bronze/Silver アクセス → 拒否を確認

```sql
-- Bronze へのアクセス（PERMISSION_DENIED になること）
SELECT * FROM northwind_catalog.bronze.orders LIMIT 1;
```

```sql
-- Silver へのアクセス（PERMISSION_DENIED になること）
SELECT * FROM northwind_catalog.silver.customers LIMIT 1;
```

期待エラー:
```
[INSUFFICIENT_PERMISSIONS] Insufficient privileges: User does not have USE SCHEMA on Schema 'northwind_catalog.bronze'. SQLSTATE: 42501
```

### Step 3-3: Gold アクセス → 正常動作を確認

```sql
-- Gold へのアクセス（成功すること）
SELECT * FROM northwind_catalog.gold.sales_by_product LIMIT 5;
SELECT * FROM northwind_catalog.gold.order_summary LIMIT 5;
```

期待結果: 各 5 行のデータが表示される。

### Step 3-4: スクリーンショット取得（証跡）

以下の 2 点を取得して `証跡/Epic2/` に保存する。

| 証跡 | 内容 |
|-----|------|
| 証跡1 | Bronze アクセス時の `PERMISSION_DENIED` エラー画面 |
| 証跡2 | Gold アクセス時の正常表示画面（データが表示されていること） |

## 注意事項

| 項目 | 内容 |
|------|------|
| admin 権限 | Phase 2 の SQL は Unity Catalog の metastore admin または catalog owner で実行すること |
| テストユーザー権限 | テストユーザーは workspace の通常ユーザーとして招待（admin 権限不要） |
| デフォルト拒否 | Unity Catalog は「付与されていない = 拒否」のため、bronze/silver に REVOKE は不要 |
| クリーンアップ | テスト完了後、テストユーザーをグループから除外または削除して良い |
